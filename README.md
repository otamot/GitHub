# GitHubについて
## Commitのメッセージ
### メッセージの書き方
* 1行目:変更内容の要約
* 2行目:空行
* 3行目:変更の理由

### 変更内容の要約
* fix：バグ修正
* hotfix：クリティカルなバグ修正
* add：新規（ファイル）機能追加
* modify：機能修正（バグではない）
* change：仕様変更
* clean：整理（リファクタリング等）
* disable：無効化（コメントアウト等）
* remove：削除（ファイル）
* upgrade：バージョンアップ
* revert：変更取り消し

### ライト版
* fix：バグ修正
* add：新規（ファイル）機能追加
* modify：機能修正（バグではない）
* remove：削除（ファイル）
